MVP stage! => * focus points for v1.0



strengths
- multirepo management via clean CLI
- rich Markdown output with color & proper alignment
- generics-based reusable table generation
- leverage established tools
- commands handle both single-repo & recursive directory scanning

weaknesses
* update table / report rendering
* sequential processing of repos makes large scans slow
* describe does not include the remote URL
* stats => bug on type casting
- stats command promises quarterly aggregation but only computes yearly => --time flag unused

usefulness
- useful for developers managing handfuls of local Git clones who need quick overviews, batch operations & lightweight repo stats

enhancements
* parallelization
* use recursive reading on children repos
* add dry-run mode
- add simple unicode graphics
- implement quarterly / aggregation logic
- consider tui for version 2.0
